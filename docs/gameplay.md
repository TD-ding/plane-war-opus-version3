# 玩法说明

经典纵版飞机射击游戏：操控己方战机在屏幕底部区域走位，躲避并击落不断下落的敌机。击毁敌机得分，被敌机撞击或被敌弹击中则扣血，血量耗尽即游戏结束。打得越连贯、越久，难度与回报都越高。

本文所有数值均来自 `index.html` 中的 `CONFIG` 配置，可在该处统一调校。

## 操作按键

| 操作 | 按键 |
| --- | --- |
| 移动 | **方向键** 或 **W A S D**，或触屏 / 鼠标拖动飞机 |
| 射击 | **自动连续开火**；按住 **空格** 触发更快的手动加射 |
| 暂停 / 继续 | **P** 或 **Esc** |
| 释放清屏炸弹 | **B**（需库存 > 0 且游戏进行中） |
| 静音 / 取消静音 | **M** |
| 开始 / 重新开始 | **回车（Enter）**，或点击屏幕 |

> 移动端友好：直接用手指拖动飞机即可移动，飞机会平滑跟随指针。

## 各系统

### 火力成长系统

子弹的发数随**火力等级**成长，等级范围 1 ~ 4：

- 1 级：单发直射。
- 2 / 3 / 4 级：每升一级多打一发，并向两侧张开角度（相邻两发约 9°），形成扇形散射，火力越猛覆盖越广。
- **升级**：拾取「强化火力」道具时火力等级 +1（已封顶 4 级则不再增加）。
- **降级**：被敌机撞击或被敌弹命中时火力等级 -1，**但下限为 1**（始终至少能打出单发，避免陷入打不出子弹的死局）。

自动开火间隔约 220ms；按住空格的手动加射间隔约 120ms，更快。

### 连击系统

连续击毁敌机会累积**连击数**，得分按倍率翻倍以奖励连贯操作：

| 连击数 | 得分倍率 |
| --- | --- |
| 0 ~ 4 | ×1 |
| 5 ~ 9 | ×2 |
| 10 ~ 19 | ×3 |
| ≥ 20 | ×4（封顶） |

- 倍率取「当前连击数已达到的最高档」。
- **断连**：约 2.2 秒内没有新的击毁，或被击中（撞机 / 中弹），连击立即清零。
- HUD 实时显示当前连击数与倍率。

### 道具系统

敌机被击毁后有约 16% 概率掉落道具，道具向下飘落，飞过去碰到即可拾取。道具存活约 7 秒，临近消失（剩约 2 秒）会闪烁预警。掉落类型按权重随机：

| 道具 | 图标 | 效果 | 权重 |
| --- | --- | --- | --- |
| 强化火力 | ▲ | 火力等级 +1 | 3 |
| 护盾 | ◈ | 一段时间内免疫碰撞与敌弹伤害 | 2 |
| 回血 | + | 恢复 1 点生命值 | 2 |
| 清屏炸弹 | ✸ | 存入库存（最多 3 个），按 B 释放 | 1 |

#### 清屏炸弹

拾取「清屏炸弹」会存入库存（上限 3 个）。被敌机包围时按 **B** 一键清空全屏敌机与敌弹，每架被清掉的敌机照常计分并产生爆炸特效。

#### 护盾

拾取「护盾」后约 6 秒内被撞 / 被打不掉血；期间飞来的敌弹会被**当场挡掉爆炸**（不再幽灵般穿身而过），并发出金属「叮」声反馈。护盾与下文「无敌帧」机制不同：护盾由道具赋予、持续较久；无敌帧是受击后的短暂缓冲。

### 敌机与敌机射击

- 敌机从屏幕顶部生成，向下掉落并轻微左右摆动。
- 尺寸随机；尺寸偏大的算「大敌机」，血量翻倍（2 血 vs 1 血），得分更高（30 分 vs 10 分）。
- **大敌机会朝玩家当前方向发射粉色子弹**，需要走位躲避。
- 屏幕上同时存在的敌机数量有上限（12 架），避免后期糊成「一堵墙」必死。

### Boss

当前版本**没有独立的 Boss 战**。后期的压力来自「大敌机血厚、会射击」以及随时间递增的刷怪密度与速度，而非单独的 Boss 单位。

### 生命、得分与音效

- 生命值 5 点，HUD 实时显示；被撞或中弹扣 1 点，归零即游戏结束。
- 实时得分 + 历史最高分（用 localStorage 持久化，并对被篡改 / 写脏的值做容错归一）。
- 程序化音效：射击、命中、爆炸、被撞、拾取、炸弹、护盾挡弹等全部用 WebAudio 实时合成，**无任何外部音频文件**，可按 M 静音。
- 视觉反馈：粒子爆炸、屏幕震动、击毁飘字「+分数」、滚动星空背景。

## 核心机制

这几条是「容易写错且值得回归保护」的核心，已抽到 `src/logic.js` 并被单元测试覆盖（见 [部署与运行](deployment.md)）。

### 帧率无关（timeScale）

游戏内所有「每帧位移量」都以 60fps 为基准设计。不同刷新率（60 / 120 / 144Hz）单帧耗时 `dt` 不同，因此用 `timeScale = dt / 基准帧间隔` 把每帧步进换算成相对基准帧的比例：

- 60fps 单帧 ≈ 1，
- 144Hz 单帧 ≈ 0.42，
- 30fps 单帧 ≈ 2。

所有位移按该比例缩放，从而在任意刷新率下游戏速度保持一致。主循环还会对异常大的 `dt`（如切后台回来）做钳制，避免一帧跳太远。

### 无敌帧（受击缓冲）

玩家被撞 / 中弹后会获得约 1.2 秒的**无敌时间**，期间不再扣血。这避免了在密集敌机里「一帧之内被多架敌机连续扣血瞬间团灭」。无敌期间飞机会闪烁，给出「当前无敌」的视觉提示。无敌结束后才会重新参与碰撞判定。

> 无敌帧（受击后短暂缓冲）与护盾（道具赋予、持续约 6 秒）是两套独立机制，但都会让玩家在生效期间免疫伤害；敌弹在两者生效时都会被挡掉爆炸而非穿身。

### 判定盒（hitbox 缩放）

碰撞用 AABB（轴对齐矩形）以「中心点距离 vs 半宽之和」判断是否相交。为了手感，玩家的**实际命中范围只取机身中间一小块**（宽高各取约 45%，`hitboxScale = 0.45`），机翼空隙不算命中，让「擦边」更宽容；敌机则用完整尺寸参与判定。这样既不会因为机翼像素而被误判，又能保证真正撞上时一定命中。
